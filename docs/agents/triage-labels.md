# Triage labels

Use these GitHub labels for the canonical triage roles:

| Role | Label |
| --- | --- |
| Needs maintainer evaluation | `needs-triage` |
| Waiting on reporter or missing information | `needs-info` |
| Fully specified and ready for an AFK implementation agent | `ready-for-agent` |
| Requires human action or local hardware/manual acceptance | `ready-for-human` |
| Will not be actioned | `wontfix` |

Use these supporting labels where helpful:

| Purpose | Labels |
| --- | --- |
| Work type | `type:feature`, `type:spike`, `type:bug` |
| Area | `area:cli`, `area:link`, `area:mcp`, `area:docs` |
| Priority | `priority:p0`, `priority:p1` |
| Scope | `post-mvp` |

Do not create duplicate labels with different spelling unless the repo's label vocabulary is intentionally changed first.
